ISSUES IDENTIFIED
================================================================================

1. ISSUE: NLP Pattern Matching Limitations (ask command)
   Severity: MEDIUM
   Impact: Users may get incorrect results when using natural language queries
   
   Details:
   - "What functions are in the codebase?" fails with "Node not found: what"
   - "show me all functions" translates to "type:Class" instead of "type:Function"
   - Pattern matching appears to have poor training or limited patterns
   
   Recommendation:
   - Improve NLP pattern matching with better training data
   - Add more pattern variations for common queries
   - Provide better error messages guiding users to use GQL instead
   - Consider documenting known limitations in help text

2. ISSUE: Blast Radius Returns 0 Callers
   Severity: MEDIUM
   Impact: blast-radius command doesn't show impact analysis results
   
   Details:
   - Both 'authenticate' and 'fetch_user' show 0 direct callers
   - In the test code, 'main' calls 'authenticate', so should show 1 caller
   - Impact zone and data-flow depth also show 0
   
   Recommendation:
   - Investigate how call edges are being stored/indexed
   - Verify that the graph topology includes call relationships
   - Test with larger/different codebases to see if issue is specific to test case
   - Review blast-radius algorithm for potential bugs

Following the audit findings, the 'ask' command has been DISABLED due to
NLP pattern matching accuracy issues identified in tests 4 and 5.

Changes Made:
-------------
1. Commented out Ask command variant in Commands enum
2. Commented out Ask match arm implementation  
3. Removed 'ask' from error message showing available commands
4. Updated "Next steps" message after discovery to remove ask reference
5. Updated CLI_Structure.txt to move 'ask' to disabled commands section

Reason for Removal:
------------------
The NLP pattern matching in the 'ask' command had significant accuracy issues:
- Query "What functions are in the codebase?" failed with "Node not found: what"
- Query "show me all functions" incorrectly translated to "type:Class" instead of "type:Function"
- Pattern matching confidence was unreliable (88% confidence with wrong results)

Recommendation:
--------------
Users should use the 'gql' command instead for querying the graph. GQL provides:
- Precise, reliable query results
- Standard graph query language syntax (MATCH ... RETURN ...)
- Better performance without NLP translation overhead
- Execution plan visualization with --explain flag
- Built-in query macros via --macro-name flag

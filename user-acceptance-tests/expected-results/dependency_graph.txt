---

  Path: user-acceptance-tests/test-data/puppet-sample/modules/webserver

    - common

  Path: user-acceptance-tests/test-data/puppet-sample/modules/database

  1. common
  2. firewall

- "firewall" appears as external dependency (not in modules/)
- Order may vary for database/firewall (both depend only on common)

## Test: Topological sort validation

Expected topological orders (all valid):
1. common, firewall, database, webserver
2. common, database, firewall, webserver
3. firewall, common, database, webserver

All three are valid because:
- common has no dependencies (can be first)
- firewall has no dependencies (can be anywhere before webserver)
- database only depends on common
- webserver depends on both common and firewall

Key constraint: common must come before database and webserver
